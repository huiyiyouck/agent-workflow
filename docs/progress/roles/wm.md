## 2026-05-30 — [基线修正提案] 入口层模糊意图反问机制

### 问题

1. 用户说"你是这个项目的管理者""你是VM"等模糊表述时，CLAUDE.md 入口层第 28 行"如果用户意图不明确，先按普通模式响应"导致直接沉到普通模式，用户连续三次才穿透入口层。
2. 精确触发列表有缺口："你是 Developer（开发工程师）"在列表里，但"你是 PM""你是 Architect""你是 WM"等同等句式不在列表里，导致"你是PM"这种明显意图反而要走模糊反问通道。

### 根因

当前入口层只有二元分流（明确触发 → 团队模式 / 其余 → 普通模式），缺中间态"可能触发但不够明确 → 反问确认"。

### 建议方案

入口层分流改为**三层**：精准触发表 → 模糊反问 → 普通模式。

#### 第一层：精准触发（直接进入团队模式，不反问）

| 触发句式 | 示例 | 动作 |
|----------|------|------|
| "进入团队模式" / "进入一人公司开发团队模式" / "使用团队工作流" | — | 直接进入 |
| "执行 Bootstrap 初始化流程" | — | 直接进入 |
| "启动标准迭代" / "创建 PRD" / "进入 Review" / "执行收尾归档" / "执行迭代关闭检查" / "执行流程审计" | — | 直接进入 |
| "你是 {角色英文名}" | 你是 PM / Developer / Tester / Architect / UI / DevOps / WM | 直接进入，加载对应角色 |
| "你是 {角色中文名}" | 你是产品经理 / 开发工程师 / 测试工程师 / 架构师 / 界面设计师 / 运维部署工程师 / 工作流管理者 | 直接进入，加载对应角色 |
| "以 {角色} 角色工作" | 以 PM 角色工作 / 以产品经理角色工作 | 直接进入，加载对应角色 |
| "你来当 {角色}" | 你来当 PM / 你来当测试 | 直接进入，加载对应角色 |

#### 第二层：模糊反问（命中关键词但不够精准时反问确认）

| 关键词 | 歧义来源 | 反问 |
|--------|----------|------|
| "管理者" / "管理" | PM 和 WM 都涉及管理 | "你说的是 **PM（产品经理）** 还是 **WM（工作流管理者）**？" |
| "产品" | 可能是角色指定，也可能只是讨论产品 | "你想以 **PM（产品经理）** 角色进入团队模式吗？" |
| "开发" / "写代码" / "程序员" / "全栈" / "前端" / "后端" | 可能是角色指定 | "你想以 **Developer（开发工程师）** 角色进入团队模式吗？" |
| "测试" / "QA" | 可能是角色指定 | "你想以 **Tester（测试工程师）** 角色进入团队模式吗？" |
| "架构" | 可能是角色指定 | "你想以 **Architect（架构师）** 角色进入团队模式吗？" |
| "UI" / "界面" / "设计" | UI 和 Architect 都可能涉及设计 | "你说的是 **UI（界面设计师）** 还是 **Architect（架构师）**？" |
| "运维" / "部署" / "发布" | 可能是角色指定 | "你想以 **DevOps（运维/部署工程师）** 角色进入团队模式吗？" |
| "VM"（笔误） | 可能是 WM | "你是说 **WM（工作流管理者）** 吗？" |

反问格式（一次给出，不多轮追问）：
```
检测到 [关键词]，你想：
1. 以 [角色] 进入团队模式
2. 暂不进入，继续普通模式
```

#### 第三层：普通模式（都不命中 → 普通模式）

### 影响范围

- `CLAUDE.md`："团队模式触发" 段重写为三层结构
- 不影响 runtime.md、角色手册、模板

### 执行结果

已修改 `CLAUDE.md`，将原"团队模式触发"列表重写为三层分流结构：精准触发表 → 模糊反问 → 普通模式。全部 7 个角色的中英文名均覆盖精准触发。

### 下一步
